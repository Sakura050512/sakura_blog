## 什么是云计算？
+ 云计算是一种**按使用量付费**的模式，这种模式提供可用的、便捷的、按需的网络访问，进入可配置的计算资源共享池（资源包括网络、服务、存储、应用软件、服务），这些资源能够被快速提供，只需投入很少的理性工作，或与服务供应商进行很少的交互。也就是说，**云计算提供的是一种新的资源按需租用的服务模式，也是一种新型的互联网数据中心业务**。
+ 简单来说：按需购买，按量付费的互联网资源。
## 云计算有什么特性？
+ **超大规模**：公有云平台动辄就有几十万台甚至上百万台服务器，企业私有云一般拥有数百上千台服务器。
+ **高可靠性**：“云”使用了数据多副本容错、计算节点同构可互换等措施，提高数据的高可靠性。
+ **多租户隔离**：允许多个不同的租户共享底层的硬件资源，但是在上层逻辑上是隔离的。
+ **弹性伸缩**：可根据业务负载的变化自动或手动调整资源规模，满足应用和用户规模增长的需求。
+ **按需服务**：用户可以根据自己的需求随时获取/使用云计算资源，无需人工干预。
+ **资源可监控计量**：云平台提供资源的监控和计量功能，用户只需为实际使用的资源付费，无需承担传统IT架构中硬件采购、机房维护等固定成本。
+ **低成本**：用户无需负担高昂的数据中心管理成本，以较低成本、最短时间获得最有效的资源，提供了极高的性价比。
+ **虚拟化**：支持用户在任意位置，使用各种终端获取应用服务。
## 云计算的**三层核心服务**
+ IaaS：基础设施即服务（提供基础设施资源）
+ PaaS：平台即服务（提供平台）
+ SaaS：软件即服务（提供应用程序）
 ![[Pasted image 20260301161806.png]]
+ 灵活性：IaaS > PaaS > SaaS
+ 成本：IaaS > PaaS > SaaS
## 云计算中有哪些计费方式？
+ **按需付费**：用户根据自己的实际使用情况付费，使用多少就付多少，价格相对较高。
+ **包年包月**：用户一次性购买一定时间内的计算资源，费用相对较低。
+ **预留实例**：海外云厂商(AWS、Azure)的主流长期计费模式，用户预付一定费用购买特定规格实例的长期使用权。
+ **按竞价方式计费**：云厂商针对闲置物理资源推出的低价计费模式，用户通过“竞拍”方式获取资源，价格远低于按需实例付费。适合可中断、非核心的业务场景。
## 云计算的四种模式以及它们之间的区别
### 四种模式
+ **公有云**：由第三方云服务商提供，通过互联网共享基础设施和服务**
	+ 核心特性：按需付费、即开即用、资源弹性扩展，安全性依赖服务商的保障机制，无需企业自建基础设施。
+ **私有云**：由企业或者第三方建设和管理，由单一云服务用户专享**（可能部署在企业内部或者托管数据中心，具备公有云所有关键特征，但硬件、软件、网络资源由该组织独占使用。）
	+ 核心特征：企业对资源的完全自主控制能力、较高的安全性水平、支持深度定制、技术灵活性高，但是需要企业自行维护基础设施且成本投入较高。
+ **混合云**：结合公有云和私有云**
	+ 核心特性：整合两者优势，关键数据存储在私有云，利用公有云实现弹性资源扩展，部署复杂度较高。
+ **==行业云==：由行业内或某个区域内起主导作用或者掌握关键资源的组织建立和维护**（如金融云、政务云、医疗云、卫生云等）
### 四种模式的区别
| 模式      | 典型场景                        | 业务价值                    | 客户关注点                   | 优势                 | 劣势                |
| :------ | :-------------------------- | :---------------------- | :---------------------- | :----------------- | :---------------- |
| **公有云** | 中小企业、电商促销、个人云存储 (如阿里云、AWS)  | 零硬件投入、快速扩容、降低门槛         | 成本、快速上线、数据主权与合规认证       | 规模大、成本低、功能全面       | 存在性能波动风险、定制化能力弱   |
| **私有云** | 金融核心系统、政府涉密平台 (如 OpenStack) | 数据本地化、满足监管、避免多租户安全风险    | 安全与隐私要求高、自主可控、运维能力评估    | 支持定制、不影响现有 IT 流程   | 成本高、周期长、扩展灵活性差    |
| **混合云** | 核心数据在私有云，报表分析或弹性扩容在公有云      | 平衡成本与安全、敏感业务本地化、非敏感业务云化 | 兼顾合规与效率、关注跨云同步与网络设计     | 架构综合、部署灵活、数据本地化    | 架构复杂、成本较高、运维难度大   |
| **行业云** | 金融云、政务云、医疗云、卫生云             | 深度适配行业流程、提供垂直方案、减少重复建设  | 行业专属合规 (如等保、医疗隐私)、系统适配性 | 针对行业优化、成本较低、共享最佳实践 | 支持范围窄、跨行业复用性差、更新慢 |
## 公有云和私有云的对比
| 功能             | 私有云                             | 公有云           |
| :------------- | :------------------------------ | :------------ |
| **服务提供**       | 根据客户的需要提供                       | 由服务提供商设置      |
| **运维管理**       | 负责运维管理的是云服务提供商、客户自己或第三方，或是上述的组合 | 由云服务提供商负责运维管理 |
| **安全管理、监控与认证** | 由云服务提供商、客户自己或第三方，也或者是它们的不同组合    | 由云服务提供商负责安全管理 |
| **弹性与可扩展**     | 可扩展，但受计算资源规模的限制                 | 无限可扩展，有服务等级保证 |
| **准备时间**       | 需要初始化建立                         | 数分钟           |
## 私有云的三种部署方式
1. **本地部署私有云**：部署在企业自有的数据中心内。
	+ 优点：控制力强，数据完全留在内部
	+ 缺点：前期资本投入高，需要专业的运维团队
2. **托管私有云**：部署在第三方服务商(如电信运营商、IDC)的机房内
	+ 优点：无需自建机房
	+ 缺点：不如本地部署直接
3. **专有云**：这是公有云厂商提供的一种服务，将公有云软硬件一体机放到本地的数据中心，提供和公有云一致的操作体验和API
	+ 优点：与公有云无缝混合，使用相同的工具和技术栈
	+ 缺点：成本较高，且与特定云厂商深度绑定
## 云计算的优缺点
### 优点
1. 对于消费者
	+ 降低前期投入成本和日常使用成本
	+ 提高了系统的可靠性、安全性、用户体验
	+ 使用户可以专注于自己的核心业务和市场
	+ 快速响应对计算资源的弹性需要
2. 对于社会
	+ 降低能耗、减少排放
	+ 提高IT设备使用率，降低设备数量，信息技术产业进一步分工合理
	+ 有利于全社会共享信息，打破信息孤岛
### 缺点
+ 严重依赖网络
+ 数据可能泄露的环节增多（安全性）
+ 用户对技术和数据的掌控度下降
## 云计算的演进
1. **积累阶段**（1959——1999）：虚拟化、网络、分布式、并行等技术成熟。
2. **初步阶段**（2006——2009）：Saleforce成立，成为全球最早的一家SaaS公司；同年Loudcloud成立，成为全球第一家IaaS服务商。
3. **形成阶段**（2009——2015）：IaaS、PaaS和SaaS三种模式全部出现，IT/电信/互联网行业纷纷进军云计算领域。
4. **发展阶段**（2009——2015）：云的种类逐渐完善，传统企业开始上云。
5. **现今阶段**（2015——）：政府和公共服务也逐渐上云，云的产品和功能逐渐健全，形成了云的标准化。
